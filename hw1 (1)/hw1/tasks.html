Homework 1: Tasks for dl324

Your personal task is to develop a system for libraries and their books.

Task 1: Configuration [4 marks]

Configure the project to support the use of JSPs as view technology. JSPs must be configured to be placed inside folder src/main/webapp/WEB-INF/views/.

Task 2: Domain Classes [13 marks]

Create class co2103.hw1.domain.Book. With the following private fields:

  1. String title
  2. String author
  3. String genre
  4. int amount
Generate necessary getters and setters. [7 marks]

Create class co2103.hw1.domain.Library. With the following private fields:

  1. int id
  2. String president
  3. String website
  4. List<Book> books
Generate necessary getters and setters. [6 marks]

Task 3: Controller (simple) [5 marks]

Create controller co2103.hw1.controller.MainController [2 marks].

  1. Requests for URL "/" (POST and GET) should show the view start (the corresponding JSP already exists). [3 marks]

Task 4: Controller, View, and Form [20 marks]

Create field public static List<Library> libraries in class co2103.hw1.Hw1Application. Initialize the list inside class co2103.hw1.Hw1Application to contain one library with two books. [4 marks]

Create controller co2103.hw1.controller.LibraryController and add methods with request mappings for the following behavior:

  1. GET requests for URL "/libraries" should show view libraries/list. [2 marks]
    • Add the list of all libraries from co2103.hw1.Hw1Application to the model as attribute libraries. [2 marks]
    • Create the corresponding JSP to list the president and website of each library followed by a link to URL "/books?library={id of library}", e.g., "/books?library=3". [4 marks]
  2. Requests for URL "/newLibrary" should show view libraries/form. [3 marks] Create the corresponding JSP with a form with modelAttribute="library" and action="/addLibrary" and inputs for all fields of class Library except for books. [5 marks]
  3. POST requests for URL "/addLibrary" should add a new library to the list of all libraries in co2103.hw1.Hw1Application and redirect to /libraries. [3 marks]

Task 5: Controller, View, and Form with additional Parameter [33 marks]

Create controller co2103.hw1.controller.BookController and add methods with request mappings for the following behavior:

  1. GET requests for URL "/books" should show view books/list. [2 marks]
    • Based on request parameter library determine the corresponding library from co2103.hw1.Hw1Application. [2 marks]
    • Add the list of all books of that library to the model as attribute books. [2 marks]
    • Create the corresponding JSP for the view to list the title, author, genre, and amount of each book. [4 marks]
    • At the end of the list in the JSP add a link to URL "/newBook?library={id of library whose books are listed}", e.g., "/newBook?library=3". [4 marks]
  2. Requests for URL "/newBook" should show view books/form. [3 marks]
    • Create the corresponding JSP with a form with modelAttribute="book" and action="/addBook" and inputs for all fields of class Book. [6 marks]
    • Also add the request parameter library with the ID of the current library. [4 marks]
  3. POST requests for URL "/addBook" should add a new book to the list of books of the library identified by request parameter library and then redirect to /libraries. [6 marks]

Task 6: Validation [25 marks]

Set up a validator for class Library and make it available in controller LibraryController. [4 marks]

When adding a new library validate that

  1. The value of field id is unique within the list Hw1Application.libraries. [3 marks]
  2. The value of field president is not empty. [2 marks]
  3. The value of field website is not empty. [2 marks]
In case of validation errors go back to the form and show error messages for each field in the form.

Set up a validator for class Book and make it available in controller BookController. [4 marks]

When adding a new book validate that

  1. The value of field title is not empty. [2 marks]
  2. The value of field author is not empty. [2 marks]
  3. The value of field genre is either "Sport", "History", or "Biography". [3 marks]
  4. The value of field amount is between (including) 5 and 8. [3 marks]
In case of validation errors go back to the form and show error messages for each field in the form.
